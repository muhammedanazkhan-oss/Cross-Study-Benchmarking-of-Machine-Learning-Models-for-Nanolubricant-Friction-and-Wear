# Data dictionary â€” `nanolubricant_dataset.csv`

Each row is one experimental datapoint: a nanoparticle-containing lubricant
condition paired with its matched neat-oil (baseline) comparator, extracted from
a published, peer-reviewed source. The file contains **196 verified records from
60 studies**.

Columns ending in `_num` are cleaned numeric versions of the corresponding
free-text field and are the values consumed by the analysis code. The `cls_*`
and `oil_*` columns are convenience one-hot flags derived from the class and
base-oil fields.

## Identification and provenance

| Column | Type | Description |
|---|---|---|
| `datapoint_id` | string | Unique row identifier (`D001`, `D002`, â€¦). |
| `ref` | string | Citation key of the source study (e.g. `[9]`). |
| `study` | string | Short study label (first author, year, venue). |
| `doi` | string | DOI of the source article (attribution). |
| `source_type` | string | Provenance channel: `folder-PDF` (author-held PDF) or `web-OA` (open-access web). |
| `cof_source` | string | Location in the source of the friction value (page / table / figure). |
| `wear_source` | string | Location in the source of the wear value. |
| `quote` | string | Verbatim sentence from the source supporting the extracted values. |
| `confidence` | string | Extraction confidence: `High` (tabulated directly), `Med` (read from a figure or reconstructed), `Low` (weakly documented). |
| `notes` | string | Free-text extraction notes (pairing, optima, caveats). |

|---|---|---|

| `cof_reduction_pct` / `cof_reduction_pct_num` | string / float | Percentage COF reduction relative to baseline. |
| `wear_metric` | string | Reported wear quantity (WSD, wear-depth, volume, rate, mass-loss, â€¦). |

## Notes on modelling use

The analysis code (`nanolube/data_prep.py`) derives from these columns:

- a **nanoparticle class**, **morphology** (`near-spherical` / `1D` / `2D-layered` / `tubular`) and **tribometer family**;
- the **log response ratio** `g = ln(1 âˆ’ R/100)` for each target, valid only when the treated/baseline ratio is positive (`R < 100`);
- target-specific modelling frames after filtering to `wt%` concentration basis, High/Medium extraction confidence, a reconstructable ratio, and complete required predictors.

No model output, prediction, fitted parameter, or manuscript result is stored in
this file â€” those are regenerated by the pipeline into the (git-ignored)
`results/` directory.
